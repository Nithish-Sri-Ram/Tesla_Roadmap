## Phase 2: IoT Core Mastery

### Communication Protocols Unleashed
- **MQTT**: The backbone of IoT
- **HTTP/HTTPS**: Web-based control
- **I2C**: Device-to-device communication
- **SPI**: High-speed peripheral communication (displays, SD cards)
- **UART**: Serial communication fundamentals — the first protocol to understand
- **Bluetooth/BLE**: Short-range magic
- **CAN Bus**: Critical for automotive and multi-robot systems
- **LoRa/LoRaWAN**: Long-range, low-power IoT — ideal for outdoor and agricultural projects
- **Zigbee**: Mesh networking for smart home applications

Often ignored by beginners — don't let it be an afterthought:
- **OWASP IoT Top 10** - [owasp.org](https://owasp.org/www-project-internet-of-things/) — The standard checklist for IoT security
- Secure boot and firmware encryption on ESP32/STM32
- TLS for MQTT connections
- Safe credential storage (no hardcoded passwords in code!)

## Phase 5: Professional Embedded Engineering

*This phase bridges the gap between hobbyist and industry-level work.*

### RTOS & Bare-Metal Programming
Move beyond Arduino's simplified abstractions into how embedded systems actually work:
- **FreeRTOS** - [freertos.org](https://www.freertos.org/) — The most widely used RTOS; runs on ESP32 and STM32
- Task scheduling, interrupt handling, semaphores, and memory management
- **Bare-metal on STM32** - Direct register manipulation without an OS
- [DigiKey's FreeRTOS Series](https://www.youtube.com/playlist?list=PLEBQazB0HUyQ4hAPU1cJED6t3DU0h34bz) — Excellent free YouTube course

### STM32 & ARM Microcontrollers
The industry standard — what companies actually use in production:
- **STM32CubeIDE** - [st.com](https://www.st.com/en/development-tools/stm32cubeide.html) — Free IDE from STMicroelectronics
- HAL libraries, clock configuration, peripheral drivers
- [Phil's Lab on YouTube](https://www.youtube.com/@PhilsLab) — Best STM32 + electronics design channel for self-learners
- STM32 Nucleo boards are affordable and lab-friendly

### Embedded Linux
From Raspberry Pi scripts to production-grade embedded systems:
- **Buildroot** - Build minimal custom Linux images for your hardware
- **Yocto Project** - [yoctoproject.org](https://www.yoctoproject.org/) — Industry standard for embedded Linux distributions
- Writing Linux kernel modules and understanding device trees
- [Bootlin's Free Training Materials](https://bootlin.com/training/) — Professional-grade, completely free

### Testing & Debugging
Real engineers debug systematically — not by guessing:
- **Logic Analyzers**: Decode SPI, I2C, UART signals visually (lab has equipment)
- **Oscilloscopes**: Measure signal integrity, timing, voltage
- **JTAG/SWD Debugging**: Step through embedded code line by line using ST-Link or J-Link
- **Unit Testing on Embedded**: [Unity Test Framework](https://www.throwtheswitch.org/unity) and Google Test for C/C++ firmware
